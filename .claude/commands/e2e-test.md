---
description: "E2E 테스트 실행 및 결과 업데이트"
---

E2E 테스트를 실행하고 결과를 문서화합니다.

**작업 순서**:

1. **환경 준비 확인**
   - Supabase 로컬 DB가 실행 중인지 확인 (`supabase status`)
   - 실행 중이 아니면 `supabase start` 실행
   - 데이터베이스가 정상인지 확인 (8개 제품, 5개 카테고리)

2. **Playwright 브라우저 설치 확인**
   - `npx playwright install chromium` 실행 (이미 설치되어 있으면 스킵)

3. **E2E 테스트 실행**
   - `npm run test:e2e` 실행
   - 개발 서버는 playwright.config.ts의 webServer 설정으로 자동 시작됨
   - 모든 테스트 결과를 캡처

4. **테스트 결과 분석**
   - 통과/실패한 테스트 수 집계
   - 실패한 테스트가 있다면 에러 메시지 확인
   - 스크린샷/비디오가 생성되었는지 확인

5. **문서 업데이트**
   - `E2E_TEST_RESULTS.md` 파일을 현재 날짜와 테스트 결과로 업데이트
   - 테스트 통과율 계산
   - 발견된 이슈가 있다면 "발견된 이슈" 섹션에 추가
   - 모든 테스트가 통과했다면 "전체 테스트 통과" 표시

6. **결과 요약 출력**
   - 사용자에게 테스트 결과 요약 표시:
     - 전체 테스트 수
     - 통과한 테스트 수
     - 실패한 테스트 수
     - 테스트 실행 시간
   - E2E_TEST_RESULTS.md 파일 경로 안내

**중요**:
- 테스트 실행 중 에러가 발생하면 상세한 에러 메시지를 사용자에게 제공
- Supabase가 실행되지 않았다면 명확히 안내
- 테스트가 실패해도 결과 문서는 반드시 업데이트
