# 🚀 확장 기능 가이드

소장님 프로젝트를 더 큰 규모나 다양한 요구사항에 맞춰 확장하는 방법을 안내합니다.

## 📋 목차

- **[멀티 테넌트 아키텍처](./01-멀티-테넌트.md)** - N개 업장을 하나의 시스템으로 운영
- **[견적서 확장 기능](./02-견적서-확장.md)** - CSV 내보내기, 연락처 추가

## 🎯 확장 기능 소개

### 1. 멀티 테넌트 (Multi-Tenant)

**적용 시나리오**:
- 여러 업장이 각자 다른 제품 카탈로그를 운영
- 업장별로 브랜딩(색상/로고/레이아웃) 커스터마이징
- 하나의 Supabase DB로 여러 프론트엔드 운영

**핵심 기능**:
- 단일 DB + `tenant_id`로 데이터 논리 분리
- RLS 정책으로 테넌트별 데이터 완전 격리
- 동적 테마 시스템으로 업장별 UI 커스터마이징
- 도메인 또는 환경변수로 테넌트 식별

**적합한 경우**:
- 소규모 (5~20개 업장)
- 저~중트래픽
- 비용 최소화 필요

📖 **[상세 가이드 보기](./01-멀티-테넌트.md)**

### 2. 견적서 확장 (Quotation Extensions)

**적용 시나리오**:
- 견적서를 CSV/Excel로 내보내기
- 전화/팩스 등 회사 연락처 추가
- PDF 다운로드, 이메일 전송

**핵심 기능**:
- CSV 내보내기 (엑셀 호환)
- 환경변수 기반 연락처 관리
- 확장 가능한 템플릿 구조

**공수**:
- CSV 내보내기: 1~2시간
- 연락처 추가: 30분

📖 **[상세 가이드 보기](./02-견적서-확장.md)**

## 🔮 향후 추가 예정 확장 기능

### 3. 결제 시스템 연동 (예정)

- PG사 연동 (토스페이먼츠, 아임포트 등)

### 4. 다국어 지원 (예정)
- i18n 시스템 구축
- 언어별 제품 정보 관리
- 자동 번역 연동

### 5. 고급 검색 (예정)
- Algolia/Meilisearch 연동
- 전문 검색 (Full-Text Search)
- 필터링 및 정렬 고도화

### 6. 재고 관리 시스템 (예정)
- 실시간 재고 추적
- 자동 재고 알림
- 발주 시스템 연동

## 💡 확장 기능 선택 가이드

### 멀티 테넌트가 필요한가요?

**YES 인 경우**:
- ✅ 동일한 비즈니스 모델을 여러 업장에 적용
- ✅ 업장별 데이터는 완전히 분리되어야 함
- ✅ 중앙 집중식 관리/업데이트 필요
- ✅ 비용 효율성이 중요

**NO 인 경우**:
- ❌ 단일 업장만 운영 (현재 기본 시스템 사용)
- ❌ 각 업장이 완전히 다른 기능 필요
- ❌ 대규모 트래픽으로 독립 인프라 필요
